Комментарии  
1)Контейнер осталноваился потому что мы подали сигнал завершения основному процессу в контейнере (nginx)  
Контейнеры Docker запускают основной процесс, который остается активным на протяжении жизни контейнера. Когда основной процесс завершает выполнение контейнер останавливается.  
2)После изменения порта на listen 81, контейнер больше не обслуживает запросы на порту 80, который сопоставлен с портом 8080 хоста. Однако Docker по-прежнему перенаправляет запросы с 127.0.0.1:8080 на порт 80 внутри контейнера. Поскольку Nginx теперь слушает на порте 81, запросы на порт 8080 с хоста не достигают сервера внутри контейнера.

Комментарии  
При выполнении команды docker compose up -d, будет запущен файл docker-compose.yaml, так как по умолчанию Docker Compose ищет файл именно с именем docker-compose.yaml.
Чтобы запустить оба файла, необходимо использовать директиву x-includes, чтобы включить другой compose файл.
